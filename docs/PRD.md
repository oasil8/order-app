# 커피 주문 앱



## 1. 프로젝트 개요



### 1.1 프로젝트명

커피 주문 앱


### 1.2 프로젝트 목적

사용자가 커피 메뉴를 주문하고, 관리자가 주문을 관리할 수 있는 간단한 풀스택 웹 앱


### 1.3 개발 범위

- 주문하기 화면(메뉴 선택 및 장바구니 기능)
- 관리자 화면(재고 관리 및 주문 상태 관리)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능



## 2. 기술 스택

- 프론트엔드: HTML, CSS, 리액트, 자바스크립트
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL


## 3. 기본 사항

- 프론트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 학습 목적이므로 사용자 인증이나 결제 기능은 제외
- 메뉴는 커피 메뉴만 있음



## 4. 화면별 PRD

### 4.1 주문하기 화면

#### 4.1.1 화면 목적

고객이 메뉴를 탐색하고 옵션을 선택한 뒤 장바구니에 담아 주문을 완료하는 화면


#### 4.1.2 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 헤더 | 브랜드명(COZY), 주문하기/관리자 네비게이션 버튼 |
| 메뉴 목록 | 메뉴 카드 그리드 (가로 3열) |
| 장바구니 | 화면 하단 고정 영역, 담긴 항목·합계·주문 버튼 표시 |


#### 4.1.3 헤더

- 좌측: 브랜드명 "COZY" 텍스트
- 우측: "주문하기" 버튼(현재 화면 강조), "관리자" 버튼
- 버튼 클릭 시 해당 화면으로 전환


#### 4.1.4 메뉴 카드

각 메뉴는 카드 형태로 표시되며 다음 요소를 포함한다.

| 요소 | 내용 |
|------|------|
| 이미지 | 메뉴 이미지 (없을 경우 placeholder 표시) |
| 메뉴명 | 예: 아메리카노(ICE) |
| 가격 | 기본 가격 표시 (예: 4,000원) |
| 설명 | 한 줄 간단 설명 |
| 옵션 체크박스 | 샷 추가 (+500원), 시럽 추가 (+0원) 등 다중 선택 가능 |
| 담기 / 닫기 버튼 | 장바구니에 추가하거나 카드를 닫는 버튼 |

**메뉴 카드 상태**
- 기본 상태: 이미지, 메뉴명, 가격만 노출
- 확장 상태: 카드 클릭 시 설명·옵션·버튼 영역이 펼쳐짐
- 재고 없음 상태: 카드 비활성화 처리, 담기 불가

**옵션 선택 규칙**
- 옵션은 체크박스 방식으로 다중 선택 가능
- 각 옵션의 추가 금액은 선택 즉시 표시 가격에 반영
- 옵션 가격이 0원인 경우에도 항목 표시


#### 4.1.5 장바구니

화면 하단에 항상 표시되는 고정 영역

| 요소 | 내용 |
|------|------|
| 항목 목록 | 메뉴명 (선택 옵션) × 수량, 소계 |
| 총 금액 | 모든 항목 합산 금액 (굵게 강조) |
| 주문하기 버튼 | 클릭 시 주문 확정 및 서버에 주문 데이터 전송 |

**장바구니 동작 규칙**
- 동일 메뉴 + 동일 옵션 조합을 재담기하면 수량이 1 증가
- 동일 메뉴라도 옵션이 다르면 별도 항목으로 표시
- 장바구니가 비어 있을 때는 영역을 숨기거나 빈 상태 메시지 표시
- 주문 완료 후 장바구니 초기화


#### 4.1.6 주문 처리 흐름

1. 고객이 메뉴 카드를 클릭해 카드를 확장
2. 원하는 옵션을 체크박스로 선택
3. "담기" 버튼 클릭 → 장바구니에 항목 추가, 카드 닫힘
4. 장바구니 영역에서 항목·금액 확인
5. "주문하기" 버튼 클릭 → 서버에 주문 데이터 전송
6. 주문 접수 완료 메시지 표시 후 장바구니 초기화


#### 4.1.7 데이터 요구사항

**조회 (GET)**
- 메뉴 목록: 메뉴명, 가격, 설명, 이미지 URL, 재고 수량, 옵션 목록

**전송 (POST)**
- 주문 데이터: 주문 항목 배열 (메뉴 ID, 옵션 목록, 수량), 총 금액, 주문 시각


#### 4.1.8 예외 및 엣지 케이스

| 케이스 | 처리 방식 |
|--------|-----------|
| 재고 0인 메뉴 | 카드 비활성화, 담기 버튼 숨김 |
| 장바구니 비어 있는 채로 주문하기 클릭 | 버튼 비활성화 또는 안내 메시지 표시 |
| 서버 오류 | 오류 메시지 표시, 장바구니 유지 |


### 4.2 관리자 화면

#### 4.2.1 화면 목적

관리자가 실시간 주문 현황을 파악하고, 재고를 조정하며, 주문 상태를 단계별로 처리하는 화면


#### 4.2.2 레이아웃 구성

| 영역 | 설명 |
|------|------|
| 헤더 | 브랜드명(COZY), 주문하기/관리자 네비게이션 버튼 |
| 대시보드 요약 | 주문 건수 현황 요약 (총 주문 / 주문 접수 / 제조 중 / 제조 완료) |
| 재고 현황 | 메뉴별 재고 수량 조회 및 수동 조정 |
| 주문 현황 | 접수된 주문 목록 및 상태 변경 버튼 |


#### 4.2.3 헤더

- 좌측: 브랜드명 "COZY" 텍스트
- 우측: "주문하기" 버튼, "관리자" 버튼(현재 화면 강조)
- 버튼 클릭 시 해당 화면으로 전환


#### 4.2.4 관리자 대시보드 요약

화면 상단에 주문 건수 현황을 한눈에 표시

| 항목 | 설명 |
|------|------|
| 총 주문 | 전체 주문 건수 |
| 주문 접수 | 접수되었으나 제조 시작 전인 건수 |
| 제조 중 | 현재 제조 진행 중인 건수 |
| 제조 완료 | 제조가 완료된 건수 |

- 수치는 서버에서 실시간으로 불러오며 주문 상태 변경 시 즉시 갱신


#### 4.2.5 재고 현황

메뉴별 재고 수량을 카드 형태로 표시

| 요소 | 내용 |
|------|------|
| 메뉴명 | 예: 아메리카노(ICE), 아메리카노(HOT), 카페라떼 |
| 현재 재고 수량 | 숫자로 표시 (예: 10개) |
| + / - 버튼 | 클릭 시 재고 수량 1 증가 / 1 감소, 서버에 즉시 반영 |

**재고 조정 규칙**
- 재고는 0 미만으로 내려갈 수 없음 (- 버튼 비활성화 또는 무시)
- 수량 변경은 즉시 서버에 저장 (별도 저장 버튼 없음)
- 재고가 0인 메뉴는 주문하기 화면에서 자동으로 비활성화


#### 4.2.6 주문 현황

접수된 주문 목록을 시간 순서대로 표시

| 요소 | 내용 |
|------|------|
| 주문 시각 | 월 일 시:분 형식 (예: 7월 31일 13:00) |
| 주문 내용 | 메뉴명 × 수량 (예: 아메리카노(ICE) x 1) |
| 주문 금액 | 해당 주문의 총 금액 (예: 4,000원) |
| 상태 버튼 | 현재 주문 상태에 따라 다음 단계로 전환하는 버튼 |

**주문 상태 흐름**

```
주문 접수 → 제조 중 → 제조 완료
```

| 현재 상태 | 버튼 레이블 | 클릭 결과 |
|-----------|-------------|-----------|
| 주문 접수 | "주문 접수" | 클릭 시 "제조 중"으로 상태 변경 |
| 제조 중 | "제조 중" | 클릭 시 "제조 완료"로 상태 변경 |
| 제조 완료 | "제조 완료" | 더 이상 변경 없음 (버튼 비활성화) |

- 상태 변경 시 대시보드 요약 수치도 함께 갱신
- 상태는 되돌릴 수 없음 (단방향 전환)


#### 4.2.7 데이터 요구사항

**조회 (GET)**
- 주문 목록: 주문 ID, 주문 시각, 주문 항목(메뉴명·옵션·수량), 총 금액, 현재 상태
- 재고 목록: 메뉴 ID, 메뉴명, 현재 재고 수량

**수정 (PATCH)**
- 주문 상태 변경: 주문 ID, 변경할 상태값
- 재고 수량 변경: 메뉴 ID, 변경된 재고 수량


#### 4.2.8 예외 및 엣지 케이스

| 케이스 | 처리 방식 |
|--------|-----------|
| 재고 수량 0에서 - 버튼 클릭 | 버튼 비활성화 또는 요청 무시 |
| 제조 완료 상태에서 상태 변경 시도 | 버튼 비활성화 |
| 서버 오류로 상태/재고 변경 실패 | 오류 메시지 표시, 이전 값으로 UI 복원 |
| 주문이 없는 경우 | 주문 현황 영역에 "접수된 주문이 없습니다" 표시 |

## 5. 백엔드 PRD


### 5.1 데이터 모델


#### 5.1.1 Menus (메뉴)

커피 메뉴 정보를 저장하는 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 메뉴 고유 ID |
| name | VARCHAR(100) NOT NULL | 메뉴 이름 (예: 아메리카노(ICE)) |
| description | TEXT | 메뉴 설명 |
| price | INTEGER NOT NULL | 기본 가격 (원 단위 정수) |
| image_url | VARCHAR(255) | 메뉴 이미지 URL |
| stock | INTEGER NOT NULL DEFAULT 0 | 현재 재고 수량 (0 이상) |
| created_at | TIMESTAMP DEFAULT NOW() | 생성 일시 |


#### 5.1.2 Options (옵션)

메뉴에 추가 가능한 옵션 정보를 저장하는 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 옵션 고유 ID |
| menu_id | INTEGER REFERENCES menus(id) | 연결된 메뉴 ID (외래 키) |
| name | VARCHAR(100) NOT NULL | 옵션 이름 (예: 샷 추가) |
| price | INTEGER NOT NULL DEFAULT 0 | 옵션 추가 금액 (원 단위 정수) |


#### 5.1.3 Orders (주문)

고객이 완료한 주문 정보를 저장하는 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 주문 고유 ID |
| total_price | INTEGER NOT NULL | 주문 총 금액 |
| status | VARCHAR(20) NOT NULL DEFAULT '주문 접수' | 주문 상태 (주문 접수 / 제조 중 / 제조 완료) |
| created_at | TIMESTAMP DEFAULT NOW() | 주문 일시 |


#### 5.1.4 Order_Items (주문 항목)

주문에 포함된 개별 메뉴 항목을 저장하는 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 항목 고유 ID |
| order_id | INTEGER REFERENCES orders(id) | 연결된 주문 ID (외래 키) |
| menu_id | INTEGER REFERENCES menus(id) | 주문된 메뉴 ID (외래 키) |
| quantity | INTEGER NOT NULL | 주문 수량 |
| item_price | INTEGER NOT NULL | 항목 단가 (기본 가격 + 옵션 가격) |


#### 5.1.5 Order_Item_Options (주문 항목 옵션)

주문 항목에 선택된 옵션을 저장하는 테이블

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | SERIAL PRIMARY KEY | 고유 ID |
| order_item_id | INTEGER REFERENCES order_items(id) | 연결된 주문 항목 ID (외래 키) |
| option_id | INTEGER REFERENCES options(id) | 선택된 옵션 ID (외래 키) |


#### 5.1.6 테이블 관계

```
menus ──< options
menus ──< order_items >── orders
order_items ──< order_item_options >── options
```


### 5.2 사용자 흐름과 데이터 연동

| 단계 | 사용자 행동 | 데이터 흐름 |
|------|------------|------------|
| ① | 주문하기 화면 진입 | Menus + Options 조회 → 메뉴 카드 표시 (재고 수량은 관리자 화면 전용) |
| ② | 메뉴 선택 및 옵션 체크 후 담기 | 클라이언트 장바구니 상태에 저장 (서버 미전송) |
| ③ | 주문하기 버튼 클릭 | Orders + Order_Items + Order_Item_Options 생성, Menus 재고 차감 |
| ④ | 관리자 화면 진입 | Orders + Order_Items + Menus 조회 → 주문 현황 표시 |
| ⑤ | 관리자 상태 버튼 클릭 | Orders.status PATCH (주문 접수 → 제조 중 → 제조 완료) |
| ⑥ | 관리자 재고 +/- 클릭 | Menus.stock PATCH |


### 5.3 API 설계


#### 5.3.1 메뉴 API

**GET /api/menus** — 메뉴 목록 조회

주문하기 화면 진입 시 호출. 각 메뉴의 옵션 목록을 함께 반환한다.

- 요청: 없음
- 응답 예시:
```json
[
  {
    "id": 1,
    "name": "아메리카노(ICE)",
    "description": "얼음이 가득한 시원한 아메리카노",
    "price": 4000,
    "image_url": null,
    "stock": 10,
    "options": [
      { "id": 1, "name": "샷 추가", "price": 500 },
      { "id": 2, "name": "시럽 추가", "price": 0 }
    ]
  }
]
```

**PATCH /api/menus/:id/stock** — 재고 수량 수정

관리자 화면에서 +/- 버튼 클릭 시 호출.

- 요청 바디: `{ "stock": 9 }`
- 응답: 수정된 메뉴 객체


#### 5.3.2 주문 API

**POST /api/orders** — 주문 생성

장바구니의 주문하기 버튼 클릭 시 호출. 주문 저장과 동시에 해당 메뉴의 재고를 차감한다.

- 요청 바디 예시:
```json
{
  "items": [
    {
      "menu_id": 1,
      "quantity": 2,
      "option_ids": [1],
      "item_price": 4500
    }
  ],
  "total_price": 9000
}
```
- 응답: 생성된 주문 객체 (id, status, created_at 포함)

**GET /api/orders** — 주문 목록 조회

관리자 화면 진입 시 호출. 최신 주문이 위에 오도록 내림차순 정렬.

- 요청: 없음
- 응답 예시:
```json
[
  {
    "id": 1,
    "total_price": 9000,
    "status": "주문 접수",
    "created_at": "2026-06-29T13:00:00.000Z",
    "items": [
      {
        "menu_name": "아메리카노(ICE)",
        "quantity": 2,
        "item_price": 4500,
        "options": ["샷 추가"]
      }
    ]
  }
]
```

**GET /api/orders/:id** — 특정 주문 조회

주문 ID를 전달하면 해당 주문의 상세 정보를 반환한다.

- 요청: URL 파라미터 `id`
- 응답: 단일 주문 객체 (items, options 포함)

**PATCH /api/orders/:id/status** — 주문 상태 변경

관리자 화면에서 상태 버튼 클릭 시 호출. 상태는 단방향으로만 전환된다.

- 요청 바디: `{ "status": "제조 중" }`
- 응답: 수정된 주문 객체
- 오류: 유효하지 않은 상태 전환 시 400 반환


### 5.4 예외 및 에러 처리

| 상황 | HTTP 상태 코드 | 처리 방식 |
|------|--------------|-----------|
| 존재하지 않는 메뉴/주문 ID | 404 | `{ "error": "Not found" }` 반환 |
| 재고 부족 상태로 주문 요청 | 409 | `{ "error": "재고가 부족합니다" }` 반환, 주문 미생성 |
| 유효하지 않은 상태 전환 | 400 | `{ "error": "유효하지 않은 상태입니다" }` 반환 |
| 필수 필드 누락 | 400 | `{ "error": "필수 항목이 누락되었습니다" }` 반환 |
| 서버 내부 오류 | 500 | `{ "error": "서버 오류가 발생했습니다" }` 반환 |
